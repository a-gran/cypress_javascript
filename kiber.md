# Техническое задание
## Игровой тренажёр «Распознай скам»

---

## 1. Общее описание

Консольная игра на Python, обучающая пользователей распознавать скам-сообщения в интернете. Игрок читает сообщения, принимает решения и получает обратную связь с объяснением.

---

## 2. Функциональные требования

### 2.1 Запуск и правила

- При запуске отображается приветствие и краткое объяснение правил
- Игрок видит: количество жизней, цель игры, способ ввода ответа

### 2.2 Игровые кейсы

Хранятся в виде списка словарей. Каждый кейс содержит три поля:

```python
{
    "message": "текст сообщения",
    "answer": "scam" | "safe",
    "explanation": "объяснение почему"
}
```

- Всего кейсов в базе: **не менее 15**
- За одну игру случайно выбираются **10–15 кейсов** через `random`
- Тематика: Roblox, поддержка банка, выигрыш приза, фишинговые ссылки, социальная инженерия

### 2.3 Игровой цикл

- Последовательный показ кейсов по одному
- Игрок вводит ответ: `1` — скам, `2` — не скам
- Проверка ответа (rule-based, сравнение со значением поля `answer`)
- Показ результата: верно / неверно
- Показ объяснения с указанием признака угрозы
- Обновление счёта и жизней

### 2.4 Жизни и счёт

- Начальное количество жизней: **3**
- Неверный ответ → `-1` жизнь
- Верный ответ → `+1` к счёту
- Текущий счёт и жизни отображаются после каждого шага

### 2.5 Завершение игры

Игра заканчивается при одном из условий:

- Пройдены все кейсы
- Жизни = 0

### 2.6 Финальный экран

Итог определяется по количеству правильных ответов:

| Результат | Условие |
|-----------|---------|
| Новичок | < 40% правильных |
| Осторожный пользователь | 40–70% |
| Кибердетектив | > 70% |

Отображается: счёт, количество жизней, звание, краткий итог.

### 2.7 Обработка ошибок ввода

- Некорректный ввод (не `1` и не `2`) → сообщение «Введите 1 или 2» и повторный запрос
- Программа не падает при любом вводе пользователя

```

└── README.md         # инструкция по запуску
```

---

## 5. Описание модулей

**`cases.py`** — хранит список всех кейсов, функция возвращает случайную выборку через `random.sample`

**`game.py`** — содержит функции:
- `check_answer(user_input, correct_answer)` — проверка ответа
- `update_score(is_correct, score, lives)` — обновление счёта
- `get_result_title(score, total)` — определение звания

**`ui.py`** — содержит функции:
- `show_welcome()` — экран приветствия
- `show_case(case, index, total)` — вывод кейса
- `show_feedback(is_correct, explanation)` — обратная связь
- `show_final(score, total, lives)` — финальный экран

**`main.py`** — импортирует модули, запускает игровой цикл

## 6. Признаки угроз для объяснений

Объяснения в кейсах должны ссылаться на один из признаков:

- просят пароль / логин
- срочность («немедленно», «через 10 минут»)
- незнакомый отправитель
- подозрительная ссылка
- обещание приза / подарка
- просьба перейти на сторонний сайт
- давление и запугивание
